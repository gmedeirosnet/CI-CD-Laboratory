## Harbor Robot Account Rotation

### Step 1: Create New Robot Account

```bash
# Access Harbor UI
open http://localhost:8082

# Login as admin (default: admin/Harbor12345)

# Navigate to: Projects > cicd-demo > Robot Accounts
# Click "New Robot Account"
```

**Configuration:**
- **Name**: `robot-ci-cd-demo-$(date +%Y%m%d)` (e.g., robot-ci-cd-demo-20260310)
- **Expiration**: 60 days from now
- **Permissions**:
  - ✓ Pull artifact
  - ✓ Push artifact
  - ✗ Delete artifact (not needed for CI/CD)
- **Repository**: `cicd-demo/*` (all repositories in project)

**Save the Secret:**

```bash
# Copy robot account secret to secure location
# Format: robot$cicd-demo+robot-ci-cd-demo-20260310
# Secret: [generated by Harbor]

# Example:
HARBOR_ROBOT_NAME="robot$cicd-demo+robot-ci-cd-demo-20260310"
HARBOR_ROBOT_SECRET="bGHRJ5wRDZrhKWCb4GDTZuLuS064gpxT"
```

### Step 2: Update .env File

```bash
# Edit .env file
vim .env

# Update Harbor credentials
HARBOR_ROBOT_NAME=robot$cicd-demo+robot-ci-cd-demo-20260310
HARBOR_ROBOT_SECRET=bGHRJ5wRDZrhKWCb4GDTZuLuS064gpxT

# Save and exit
```

### Step 3: Update Jenkins Credentials

```bash
# Access Jenkins UI
open http://localhost:8080

# Navigate to: Manage Jenkins > Manage Credentials
# Select: (global) domain > Add Credentials
```

**Update Existing Credential:**
- **Kind**: Username with password
- **Scope**: Global
- **ID**: `harbor-robot-credentials`
- **Username**: `robot$cicd-demo+robot-ci-cd-demo-20260310`
- **Password**: `[new robot secret]`
- **Description**: Harbor Robot Account (rotated 2026-03-10)

**Or via Jenkins CLI:**

```bash
# Update credentials via CLI
jenkins-cli update-credentials-by-xml system::system::jenkins "harbor-robot-credentials" < harbor-creds.xml
```

### Step 4: Update Kubernetes Secret

```bash
# Update Harbor secret in Kubernetes
kubectl create secret docker-registry harbor-registry-credentials \
  --docker-server=host.docker.internal:8082 \
  --docker-username="robot\$cicd-demo+robot-ci-cd-demo-20260310" \
  --docker-password="bGHRJ5wRDZrhKWCb4GDTZuLuS064gpxT" \
  --namespace=app-demo \
  --dry-run=client -o yaml | kubectl apply -f -

# Verify secret updated
kubectl get secret harbor-registry-credentials -n app-demo -o jsonpath='{.data.\.dockerconfigjson}' | base64 -d
```

### Step 5: Test New Credentials

```bash
# Test Docker login with new credentials
docker login localhost:8082 -u "robot\$cicd-demo+robot-ci-cd-demo-20260310" -p "bGHRJ5wRDZrhKWCb4GDTZuLuS064gpxT"

# Expected output: Login Succeeded

# Test push/pull
docker tag hello-world localhost:8082/cicd-demo/test:latest
docker push localhost:8082/cicd-demo/test:latest
docker pull localhost:8082/cicd-demo/test:latest
```

### Step 6: Trigger Test Build

```bash
# Trigger Jenkins pipeline to test new credentials
# Navigate to Jenkins > cicd-demo-pipeline > Build Now

# Monitor build logs for Harbor authentication
# Expected: Successful authentication and image push
```

### Step 7: Revoke Old Robot Account

**IMPORTANT: Only after successful testing**

```bash
# Access Harbor UI
open http://localhost:8082

# Navigate to: Projects > cicd-demo > Robot Accounts
# Find old robot account (e.g., robot-ci-cd-demo-20260110)
# Click "Disable" or "Delete"
```

### Rollback Procedure (if issues occur)

```bash
# Re-enable old robot account in Harbor UI
# Revert .env file to old credentials
# Update Jenkins credentials back to old values
# Update Kubernetes secret back to old credentials
```

## Jenkins Credentials Rotation

### Step 1: Generate New Admin Password

```bash
# Generate strong password
NEW_JENKINS_PASSWORD=$(openssl rand -base64 16)
echo "New Jenkins Password: $NEW_JENKINS_PASSWORD"

# Save to secure password manager
```

### Step 2: Update Jenkins Admin Password

```bash
# Access Jenkins UI
open http://localhost:8080

# Login with current password

# Navigate to: admin (top right) > Configure
# Scroll to "Password" section
# Enter new password twice
# Click "Save"
```

### Step 3: Update .env File

```bash
# Edit .env file
vim .env

# Update Jenkins password
JENKINS_PASSWORD=$NEW_JENKINS_PASSWORD

# Save and exit
```

### Step 4: Update Jenkins API Token (if used)

```bash
# Navigate to: admin (top right) > Configure
# Scroll to "API Token" section
# Click "Add new Token"
# Name: jenkins-api-token-20260310
# Click "Generate"

# Copy token and update .env
JENKINS_API_TOKEN=[generated token]
```

### Step 5: Test New Credentials

```bash
# Test Jenkins CLI with new credentials
java -jar jenkins-cli.jar -s http://localhost:8080 -auth admin:$NEW_JENKINS_PASSWORD who-am-i

# Expected output: Authenticated as: admin

# Test API access
curl -u admin:$JENKINS_API_TOKEN http://localhost:8080/api/json
```

### Step 6: Update CI/CD Scripts

```bash
# If any scripts use hardcoded credentials, update them
# Search for old credentials
grep -r "JENKINS_PASSWORD" scripts/

# Update all references to use .env file
source .env
```

## ArgoCD Password Rotation

### Step 1: Generate New Password

```bash
# Generate strong password
NEW_ARGOCD_PASSWORD=$(openssl rand -base64 16)
echo "New ArgoCD Password: $NEW_ARGOCD_PASSWORD"
```

### Step 2: Update ArgoCD Admin Password

```bash
# Method 1: Using argocd CLI
argocd login localhost:8090 --insecure
argocd account update-password --account admin \
  --current-password "$ARGOCD_ADMIN_PASSWORD" \
  --new-password "$NEW_ARGOCD_PASSWORD"

# Method 2: Using kubectl
kubectl -n argocd patch secret argocd-secret \
  -p '{"stringData": {
    "admin.password": "'$(htpasswd -nbBC 10 admin $NEW_ARGOCD_PASSWORD | cut -d: -f2)'",
    "admin.passwordMtime": "'$(date +%FT%T%Z)'"
  }}'
```

### Step 3: Update .env File

```bash
# Edit .env file
vim .env

# Update ArgoCD password
ARGOCD_ADMIN_PASSWORD=$NEW_ARGOCD_PASSWORD

# Save and exit
```

### Step 4: Test New Password

```bash
# Test login with new password
argocd login localhost:8090 --insecure \
  --username admin \
  --password "$NEW_ARGOCD_PASSWORD"

# Expected: Logged in successfully

# Test application access
argocd app list
```

### Step 5: Update Jenkins ArgoCD Credentials

```bash
# If Jenkins pipeline uses ArgoCD CLI
# Update ArgoCD password in Jenkins credentials

# Navigate to: Manage Jenkins > Manage Credentials
# Find: argocd-admin-credentials
# Update password to new value
```

### Step 6: Restart ArgoCD Server (if needed)

```bash
# Restart ArgoCD server to apply changes
kubectl rollout restart deployment argocd-server -n argocd

# Wait for restart
kubectl rollout status deployment argocd-server -n argocd
```

## PostgreSQL Password Rotation

### Step 1: Generate New Password

```bash
# Generate strong password
NEW_POSTGRES_PASSWORD=$(openssl rand -base64 16)
echo "New PostgreSQL Password: $NEW_POSTGRES_PASSWORD"
```

### Step 2: Update PostgreSQL User Password

```bash
# Connect to PostgreSQL pod
kubectl exec -it -n app-demo postgres-0 -- psql -U postgres

# Update password
ALTER USER postgres WITH PASSWORD '$NEW_POSTGRES_PASSWORD';
\q

# For application user
psql -U postgres
ALTER USER cicd_user WITH PASSWORD '$NEW_POSTGRES_PASSWORD';
\q
```

### Step 3: Update Kubernetes Secret

```bash
# Update database credentials secret
kubectl create secret generic postgres-credentials \
  --from-literal=username=cicd_user \
  --from-literal=password="$NEW_POSTGRES_PASSWORD" \
  --namespace=app-demo \
  --dry-run=client -o yaml | kubectl apply -f -

# Verify secret updated
kubectl get secret postgres-credentials -n app-demo -o jsonpath='{.data.password}' | base64 -d
```

### Step 4: Update Application Configuration

```bash
# Update Spring Boot application.properties or application.yml
# If using environment variables (recommended):
# SPRING_DATASOURCE_PASSWORD will be updated from Kubernetes secret

# If hardcoded in values.yaml, update:
vim helm-charts/cicd-demo/values.yaml

# Update:
database:
  password: $NEW_POSTGRES_PASSWORD  # Use secret reference instead!
```

### Step 5: Restart Application Pods

```bash
# Restart backend deployment to pick up new password
kubectl rollout restart deployment cicd-demo-backend -n app-demo

# Wait for rollout
kubectl rollout status deployment cicd-demo-backend -n app-demo

# Check logs for successful connection
kubectl logs -n app-demo -l app=cicd-demo-backend --tail=50
```

### Step 6: Test Database Connection

```bash
# Test connection from application
curl http://localhost:8001/actuator/health

# Expected:
# {
#   "status": "UP",
#   "components": {
#     "db": {
#       "status": "UP"
#     }
#   }
# }

# Test direct PostgreSQL connection
kubectl exec -it -n app-demo postgres-0 -- psql -U cicd_user -d cicd_db
# Enter new password when prompted
```

## Kubernetes Secrets Rotation

### Step 1: Backup Existing Secrets

```bash
# Backup all secrets in namespace
kubectl get secrets -n app-demo -o yaml > secrets-backup-$(date +%Y%m%d).yaml

# Encrypt backup file
gpg --symmetric --cipher-algo AES256 secrets-backup-$(date +%Y%m%d).yaml

# Store encrypted backup securely
```

### Step 2: Rotate Docker Registry Secret

```bash
# Create new registry secret
kubectl create secret docker-registry harbor-registry-credentials-new \
  --docker-server=host.docker.internal:8082 \
  --docker-username="[new robot account]" \
  --docker-password="[new robot secret]" \
  --namespace=app-demo

# Update deployment to use new secret
kubectl patch deployment cicd-demo-backend -n app-demo \
  -p '{"spec":{"template":{"spec":{"imagePullSecrets":[{"name":"harbor-registry-credentials-new"}]}}}}'

# Verify pods restarted successfully
kubectl get pods -n app-demo

# Delete old secret
kubectl delete secret harbor-registry-credentials -n app-demo
```

### Step 3: Rotate TLS Secrets

```bash
# Generate new TLS certificate
openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout tls-new.key -out tls-new.crt \
  -subj "/CN=cicd-demo.local/O=DevOps Lab"

# Create new TLS secret
kubectl create secret tls cicd-demo-tls-new \
  --cert=tls-new.crt \
  --key=tls-new.key \
  --namespace=app-demo

# Update ingress to use new secret
kubectl patch ingress cicd-demo -n app-demo \
  -p '{"spec":{"tls":[{"hosts":["cicd-demo.local"],"secretName":"cicd-demo-tls-new"}]}}'

# Delete old secret
kubectl delete secret cicd-demo-tls -n app-demo
```

### Step 4: Rotate Service Account Tokens

```bash
# Service account tokens auto-rotate in Kubernetes 1.21+
# Manual rotation for older versions:

# Delete service account token (will be regenerated)
kubectl delete secret $(kubectl get sa cicd-demo-app -n app-demo -o jsonpath='{.secrets[0].name}') -n app-demo

# Verify new token created
kubectl get sa cicd-demo-app -n app-demo -o yaml
```

## Automation Scripts

### Automated Rotation Script

Create `scripts/rotate-credentials.sh`:

```bash
#!/bin/bash
# Automated credential rotation script

set -euo pipefail

ROTATION_TYPE="$1"
ENVIRONMENT="${2:-.env}"

source "$ENVIRONMENT"

rotate_harbor_robot() {
    echo "Rotating Harbor robot account..."
    # Implementation here
}

rotate_jenkins_password() {
    echo "Rotating Jenkins password..."
    # Implementation here
}

rotate_sonarqube_token() {
    echo "Rotating SonarQube token..."
    # Implementation here
}

case "$ROTATION_TYPE" in
    harbor)
        rotate_harbor_robot
        ;;
    jenkins)
        rotate_jenkins_password
        ;;
    sonarqube)
        rotate_sonarqube_token
        ;;
    all)
        rotate_harbor_robot
        rotate_jenkins_password
        rotate_sonarqube_token
        ;;
    *)
        echo "Usage: $0 {harbor|jenkins|sonarqube|all}"
        exit 1
        ;;
esac

echo "Credential rotation completed successfully!"
```

### Make executable:

```bash
chmod +x scripts/rotate-credentials.sh
```

### Usage:

```bash
# Rotate specific credential
./scripts/rotate-credentials.sh harbor

# Rotate all credentials
./scripts/rotate-credentials.sh all
```

## Post-Rotation Verification

### Verification Checklist

After rotating credentials, verify all systems are working:

```bash
# 1. Test Jenkins pipeline
curl -u admin:$JENKINS_PASSWORD http://localhost:8080/api/json

# 2. Test Harbor access
docker login localhost:8082 -u "$HARBOR_ROBOT_NAME" -p "$HARBOR_ROBOT_SECRET"

# 3. Test SonarQube analysis
mvn sonar:sonar -Dsonar.login=$SONAR_TOKEN

# 4. Test ArgoCD access
argocd app list --server localhost:8090 --insecure

# 5. Test GitHub access
git ls-remote https://$GITHUB_TOKEN@github.com/$GITHUB_REPO

# 6. Test database connection
kubectl exec -it -n app-demo postgres-0 -- psql -U postgres -c "SELECT 1"

# 7. Run full deployment test
./scripts/test-deployment.sh
```

### Rollback Plan

If issues occur after rotation:

```bash
# 1. Restore from backup
kubectl apply -f secrets-backup-[date].yaml

# 2. Revert .env file
git checkout .env

# 3. Restart affected services
kubectl rollout restart deployment -n app-demo

# 4. Re-enable old credentials in external systems
# - Harbor: Re-enable old robot account
# - SonarQube: Re-generate old token (if possible)
# - GitHub: Create new token with same permissions
```
