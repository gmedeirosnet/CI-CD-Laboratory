BlackDuck Detect Stage — What Happens Inside
  1. Jenkins pulls blackducksoftware/detect:latest (or pinned version) from Docker Hub
  2. Container runs with workspace mounted as volume
  3. Detect inspects pom.xml, package.json, and the built JAR for dependency info
  4. Generates a scan report (JSON + text summary) written to blackduck-reports/ in workspace
  5. Jenkins archives blackduck-reports/** as a build artifact
  6. Stage always succeeds (audit mode) — findings are visible in Jenkins artifacts
