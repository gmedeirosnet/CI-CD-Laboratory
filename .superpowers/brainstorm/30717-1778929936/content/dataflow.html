BlackDuck Integration — Data Flow & Configuration

Credentials, environment variables, and report artifacts

Environment Variables (.env.template additions)
BLACKDUCK_URL=https://your-blackduck-server # optional — omit for offline/detect-only mode
BLACKDUCK_API_TOKEN=your-token-here        # optional — omit for offline/detect-only mode
BLACKDUCK_DETECT_VERSION=latest         # pin to a specific version e.g. 8.11.0
BLACKDUCK_PROJECT_NAME=cicd-demo        # project name in BD Hub (if connected)
BLACKDUCK_PROJECT_VERSION=${BUILD_NUMBER} # version tag per build
Jenkins Stage — Data Flow

Inputs

  • Built Docker image tag (IMAGE_TAG)
  • Workspace with pom.xml + frontend/package.json
  • Built JAR in target/
  • Optional: BLACKDUCK_URL + BLACKDUCK_API_TOKEN from Jenkins credentials

Outputs

  • blackduck-reports/detect-output.txt — console log
  • blackduck-reports/risk-report.json — structured findings
  • Jenkins archived artifact: blackduck-reports/**
  • Build description updated with component count
Jenkins Stage — Shell Script (inside Jenkinsfile)
mkdir -p blackduck-reports docker run --rm \ -v "${WORKSPACE}:/workspace" \ -v "${WORKSPACE}/blackduck-reports:/workspace/blackduck-reports" \ -e DETECT_SOURCE_PATH=/workspace \ -e DETECT_OUTPUT_PATH=/workspace/blackduck-reports \ -e DETECT_PROJECT_NAME="${BLACKDUCK_PROJECT_NAME:-cicd-demo}" \ -e DETECT_PROJECT_VERSION_NAME="${BUILD_NUMBER}" \ ${BLACKDUCK_URL:+-e BLACKDUCK_URL="${BLACKDUCK_URL}"} \ ${BLACKDUCK_API_TOKEN:+-e BLACKDUCK_API_TOKEN="${BLACKDUCK_API_TOKEN}"} \ blackducksoftware/detect:${BLACKDUCK_DETECT_VERSION:-latest} \ || true # audit mode: never fail the build archiveArtifacts artifacts: 'blackduck-reports/**', allowEmptyArchive: true
Credentials in Jenkins (optional — only if Hub is available)
  • Credential ID: blackduck-api-token (Secret Text)
  • Credential ID: blackduck-url (Secret Text)
  • Both are optional — stage runs in offline mode if absent
